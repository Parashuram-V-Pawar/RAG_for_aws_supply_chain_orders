# RAG for AWS Supply Chain Orders
## Project Overview
This project implements a Retrieval-Augmented Generation (RAG) system on top of a supply chain dataset.
It enables users to ask natural language questions about orders, warehouses, delays, and product performance—and receive context-aware answers generated by an LLM.

The system combines:
- Data preprocessing (Pandas)
- Semantic search (Sentence Transformers + FAISS)
- LLM-based response generation (Groq - LLaMA 3)

## Architecture
```
Raw CSV Data
     ↓ 
Data Preprocessing 
     ↓ 
Text Conversion (Row → Natural Language) 
     ↓ 
Embeddings (Sentence Transformers) 
     ↓ 
Vector Store (FAISS)
     ↓ 
User Query
     ↓ 
Semantic Retrieval (Top-K relevant rows) 
     ↓ 
LLM (Groq - LLaMA 3)
     ↓ 
Final Answer
```

## Features
1. Cleans and preprocesses raw supply chain data
2. Converts structured rows into semantic text
3. Uses vector embeddings for similarity search
4. Retrieves relevant records using FAISS
5. Generates answers using an LLM
6. Interactive CLI-based querying

## Setup Instructions
```
Clone the Repository
-> git clone https://github.com/Parashuram-V-Pawar/RAG_for_aws_supply_chain_orders.git

Move to project directory
-> cd RAG_for_aws_supply_chain_orders

Create Virtual Environment
-> python -m venv venv 
-> venv\Scripts\activate # Windows
-> source venv/bin/activate # Mac/Linux 

Install Dependencies
-> pip install -r requirements.txt

Configure Environment Variables
-> reate a file in src:
    .env

-> Add your Groq API key:
    GROQ_API_KEY=your_api_key_here

Run the Application
-> python main.py
```
